# 🏰 Villa Booking System: Project Overview

An elite, full-stack ecosystem designed for luxury villa rentals, featuring a high-performance React frontend and a robust Django REST API.

---

## 🏗️ System Architecture

### 1. Backend Core (The Engine)
Located in `/Backend`, this is the mission control of the application. It handles data persistence, business logic, and secure API delivery.

- **Framework**: Django & Django REST Framework (DRF)
- **Database**: PostgreSQL (Production-ready) / SQLite (Development)
- **API Strategy**: Multi-versioned architecture (`/api/v1/` and `/api/v2/`) for seamless scalability and backward compatibility.
- **Documentation**: Fully automated Swagger/OpenAPI interactive documentation.
- **Key Logic**:
    - **Dynamic Booking Engine**: Manages availability, price calculations, and booking states.
    - **Rating System**: Real-time average rating calculation for villas based on user reviews.
    - **User Management**: Integrated User Profiles and Wishlist functionality.

### 2. Frontend Surface (The Experience)
Located in `/fronted_copy`, built with cutting-edge React patterns for a fluid, "App-like" feel.

- **Framework**: React (Vite-powered)
- **Performance**: Leverages experimental features like `SuspenseList` and `ViewTransitions` for ultra-smooth navigation.
- **Quality Control**: Strict ESLint and Espree integration to ensure clean, maintainable code.
- **Styling**: Premium, responsive UI components tailored for high-end hospitality experiences.

### 3. Infrastructure & DevOps (The Foundation)
Modern deployment and automation workflows.

- **Containerization**: Full Docker support with optimized `Dockerfile` for both Backend and Frontend, orchestrated via `docker-compose.yml`.
- **CI/CD**: Advanced GitHub Actions workflows for automated testing, building, and deployment (`GITHUB_ACTIONS.md`).
- **Environment Management**: Separation of concerns with `.env` and `.env.docker` configurations.

## 🌉 The Bridge: Seamless Integration

1. **The Request**: User interacts with a React component (e.g., clicks "Add to Wishlist").
2. **The Handshake**: React dispatches a secure request to the Django API via `axios`/`fetch`.
3. **The Logic**: Django validates the session, updates the PostgreSQL database, and triggers any necessary side effects.
4. **The Response**: API returns a lightweight JSON payload.
5. **The Update**: React updates the UI state instantly without a page reload, providing a premium user experience.
