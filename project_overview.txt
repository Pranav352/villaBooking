FULL STACK PROJECT OVERVIEW: VILLA_BACKEND & FRONTEND
==========================================================================

1. BACKEND ARCHITECTURE (Django & Django REST Framework)
--------------------------------------------------------
The backend (located in /villa_backend) acts as the "Brain" and "Data Vault" of your application.

Key Files:
- settings.py: The central configuration. It manages database connections (PostgreSQL), security keys, installed apps (like 'api' and 'rest_framework'), and middleware (CORS, Authentication).
- urls.py: The routing table. It directs incoming web requests to the correct logic (e.g., any request to /api/ is sent to the api app).
- wsgi.py / asgi.py: The entry points for web servers to communicate with your code (WSGI for standard sync traffic, ASGI for modern async/WebSockets).

How it works:
The backend receives HTTP requests from the frontend, validates credentials (via Token/Session authentication), interacts with the PostgreSQL database to fetch or save data, and returns a JSON response.

Why it's important:
- Security: It keeps sensitive logic and database credentials hidden from the user.
- Data Integrity: It ensures only valid data is saved to the database.
- Centralization: Multiple frontends (Web, Mobile, IoT) can all use this single source of truth.

2. FRONTEND ARCHITECTURE (React & Tooling)
------------------------------------------
The frontend (found in /fronted_copy) is the "Face" of your application. It provides the User Interface (UI).

Key Components:
- React (Canary/Experimental): You are using high-end React features (like SuspenseList and ViewTransitions) to create smooth, high-performance user experiences.
- ESLint & Espree: These tools analyze your code as you write it. They find bugs and enforce consistent coding styles.
- Optionator: A library for parsing command-line options, likely used in your build scripts or development tools.

How it works:
React renders the UI in the user's browser. It uses "State" to remember what the user is doing and "Props" to pass data between components. It communicates with the backend by making 'fetch' or 'axios' calls to the Django API endpoints.

Why it's important:
- Interactivity: React allows parts of the page to update without a full refresh, making the app feel fast and responsive.
- Modularization: You can build small, reusable components (buttons, forms, cards) and assemble them into complex pages.

3. THE BRIDGE: HOW THEY WORK TOGETHER
-------------------------------------
The interaction follows this flow:
1. User Action: A user clicks "Book Villa" on the React frontend.
2. API Call: React sends a POST request to the Django backend (e.g., /api/bookings/).
3. CORS Check: The 'corsheaders' in Django settings checks if the frontend is allowed to talk to the backend.
4. Logic & DB: Django verifies the user, saves the booking to PostgreSQL, and sends a "Success" message.
5. UI Update: React receives the success message and updates the screen to show "Booking Confirmed."

4. WHY THIS SETUP IS USED (The "Why")
-------------------------------------
1. Separation of Concerns: Developers can work on the frontend design and backend logic independently.
2. Scalability: If the frontend gets too many users, you can scale the backend servers without changing the frontend code.
3. Reusability: The backend API can eventually serve a mobile app (iOS/Android) without rewriting the core logic.
4. Professionalism: Using tools like ESLint and Django's built-in security features ensures the code is production-ready, secure, and maintainable for years.
